# M4.5 Slice C 系统显示与辅助交互技术计划

> Revision：CT1。范围以
> [`M4.5-slice-c-polish-decision.md`](./M4.5-slice-c-polish-decision.md)
> 为准。

## 实施顺序

1. **卡片交互与字号**
   - `BiliUI` 增加窄职责 `VideoCardButtonStyle` 和无 UI 依赖的反馈策略值。
   - 热门、搜索、历史三个呈现位置复用同一 style；selected 继续来自既有返回
     快照，不新增选择 owner。
   - 标题与 footer 使用系统语义字号；hover／pressed 只改变 surface、stroke、
     opacity 和极小 scale，Reduce Motion 禁止 scale 动画。

2. **关键辅助语义与键盘**
   - `NSSearchField` 显式设置 label、help 与 identifier，Return 继续调用现有
     submit owner。
   - 卡片按钮说明“播放视频”；分 P 行合并朗读 index、title、duration 和当前项；
     登录二维码不再是完全 decorative。
   - 播放返回增加 cancel keyboard shortcut；不增加菜单 Commands 或全局媒体键。
   - 现有失败／空／加载 View 继续复用真实 ViewModel state，只补稳定 identifier
     和必要 label，不改变错误映射。

3. **Debug-only fixture 与 XCUI**
   - `BiliKitMacApp` 只在 `#if DEBUG` 且包含 `-ui-testing` 时创建
     `UITestContentView`；其他 Debug 启动和全部 Release 启动仍走 `ContentView()`。
   - fixture repository 返回自制热门／搜索／详情／多分 P，fake playback 立即
     完成；字幕为空、弹幕 presentation 无副作用、认证固定 signed-out，且所有
     identity、标题与作者均明确为测试假值。
   - launch arguments 可启用 dark 与文字放大组合；XCUI 不修改用户系统设置。
     Increase Contrast、Reduce Transparency、Reduce Motion 与 Differentiate
     Without Color 是只读系统输入，只以策略测试和真实系统成对观察验证。

## 测试与验证

- Package 测试：
  - 卡片交互策略的 hovered／pressed／selected／contrast／reduce-motion 表；
  - 既有网格、格式化、状态与播放测试保持通过。
- App tests：
  - AppKit 搜索字段的可访问 label／identifier；
  - Slice A／B route、返回、单 host 与清理测试保持通过。
- XCUI：
  - 1320×820 默认 fixture 核心路径；
  - 1080×680 深色／文字放大 fixture，检查关键元素存在、可激活且无不可达状态；
  - 截图附件只包含自制假值，成功时保留用于本机视觉核对。
- 真实签名 App：
  - 默认／紧凑窗口、自由缩放、全屏；
  - 浅／深色与必要系统辅助显示成对观察；
  - 不记录账号、真实标题、BVID、字幕／弹幕正文、完整 URL 或画面。

