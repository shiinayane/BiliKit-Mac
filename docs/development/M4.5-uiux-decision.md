# M4.5 UI/UX 决策契约

> Revision：U6。状态：用户已于 2026-07-23 确认设计方向，并确认播放页现有信息层级、未来 UP 主信息位置与单分 P 隐藏规则；本文件授权设计与验证准备，不授权生产实现。待确认的生产授权见 [`M4.5-uiux-production-decision.md`](./M4.5-uiux-production-decision.md)，App 级长期方向另见 [`../product/UIUX-VISION.md`](../product/UIUX-VISION.md)。

## 为什么做

当前 App 使用“入口侧栏—视频列表—详情播放器”三栏结构。它适合验证纵向链路，却让首页封面过小、播放页被固定列表挤压，历史与账户又分散在工具栏和 sheet。M4.5 要把已经可用的浏览、搜索、历史、播放、字幕和弹幕整理成适合日常使用的 Mac 产品，不借 UI 精修增加新的业务能力。

当前只准备设计。M4 总 Gate 及既有 resize、fullscreen、资源清理和性能基线关闭前，不得开始 M4.5 生产实现；否则无法区分旧缺口与 UI 重构回归。

## 决定什么

1. App 改为“侧栏 + 主内容”的两栏外壳。侧栏第一项是搜索页面入口，其下依次是热门与观看历史；账户入口固定在左下。选中搜索后，搜索输入框出现在主内容区顶部 toolbar，而不是嵌入侧栏。
2. 首页使用响应式卡片流：16:9 封面、最多两行标题和一行必要元信息。这里展示现有热门数据，不宣称个性化推荐。
3. 点击视频后在同一窗口进入独立播放页；返回时保留来源页面、列表滚动位置与选中卡片。
4. 宽窗口播放页使用主观看区与右栏；窄窗口允许先收起侧栏，再把右栏折叠为可展开区域，不能为了保留右栏压坏播放器。
5. 评论和相关推荐属于目标信息架构，但当前没有数据闭环。M4.5 不显示假数据或空壳；真实接入另立只读功能契约。

## 页面结构

### 首页、搜索与历史

- 侧栏是一级导航，不再用中间列表栏长期占据窗口。搜索按 Apple TV for Mac 的形态成为侧栏第一项；选中后，toolbar 显示原生搜索框，主内容区显示现有 WBI 搜索结果。
- 热门是默认浏览入口；观看历史由 sheet 迁为主内容页面。
- 搜索结果直接复用热门页的 16:9 视频卡片流与信息层级，不设置“最佳匹配”等独立结果形态。
- 搜索分类 Tab、筛选和排序属于后续功能，不在本次 UI/UX 精修中设计或实现，也不预留占位状态。
- 未输入关键词时继续使用简洁、可理解的空状态。本机搜索历史、Bilibili 热搜、分区或热门发现内容属于后续功能；本次不新增 endpoint、持久化或跨页面数据编排。
- 未登录时历史页给出可理解的登录引导；账户入口只表达“已登录／未登录”等现有安全状态，不虚构头像、昵称、等级或会员信息。
- 扫码、退出等短任务继续使用账户 sheet。

### 独立播放页

- 主观看区按“标题—UP 主／播放量／弹幕数—播放器—字幕与弹幕设置—简介”排列；元信息紧跟标题，设置紧跟播放器。
- 宽屏右栏当前优先容纳现有分 P；未来接入 UP 主资料与关注功能后，头像、名称、简介和关注按钮固定在右栏最上方，分 P 与相关推荐依次位于其下。
- UP 主资料与关注是后续真实功能。本次只确定未来信息架构，不显示假资料、不创建占位 View，也不实现关注写操作。
- 现有分 P 数据只有一项时隐藏整个分 P 模块；存在多项时才显示列表与当前项。这只改变呈现，不改变播放选择或资源生命周期。
- 评论未来接在主内容下方；M4.5 不为它制造加载态或占位区。
- 本次不加入点赞、投币、收藏、关注、分享或其他写操作。

## 响应式与 Mac 行为

- 覆盖路线图既定的 1080×680、1320×820、自由缩放和全屏。
- 首页卡片按可用宽度自适应列数；封面比例与信息高度稳定，不能为了视觉错落重排数据、键盘或 VoiceOver 顺序。
- 播放页只有清晰的主滚动关系，不嵌套多个竞争的 `List` 与 `ScrollView`。
- 使用系统语义色、材料、焦点环和 SF Symbols；支持浅色／深色、Increase Contrast、Reduce Transparency、Reduce Motion 与文字放大。
- 返回来源页面必须恢复浏览上下文；切换 route、登出和关闭窗口仍遵守已有取消、generation 与资源清理契约。

## 不做什么

- 不新增评论、相关推荐、收藏、稍后再看、推荐写操作、endpoint、持久化 schema 或个人资料字段。
- 不为评论或相关推荐创建 View、state、公开 API 或不可见占位代码。
- 不实现独立播放器窗口、mini player、PiP、媒体键或完整 Commands。
- 不照搬 B 站网页的广告、运营横幅、频道大全和密集顶部导航。
- 不创建新的 `DesignSystem` target、通用网页组件库或第二个 `Package.swift`。
- 不以视觉效果为由改动播放器、字幕或弹幕的性能与生命周期路线。

## 复杂度预算

- 决策契约不超过两页；首轮只重构三个主要表面：主导航（含历史迁移）、首页／搜索卡片呈现、播放页。
- 只允许一个 App route／浏览快照 owner；零新 target、endpoint、schema、业务 protocol 或通用布局框架。
- 只为两个以上真实调用方提取最小颜色、间距、圆角、排版和交互 token。
- 不开发通用布局框架；导航状态与滚动恢复的实现复杂度不得超过页面本身。
- 若需要新 endpoint、跨 Feature 直接依赖、新 target、长期双导航或突破 M5 边界，必须停止并重新请求用户确认。

## 完成与停止

设计完成至少需要：

- 宽屏与窄屏线框能解释首页、播放页、返回、历史和账户入口。
- 用户可区分 M4.5 会实现的现有能力重构，与评论／相关推荐的后续数据能力。
- 生产实现前另行提交技术计划、失败场景和验证矩阵。
- M4 总 Gate 与可复用性能／清理基线已经先行关闭。

若设计只能依靠假数据、空壳入口、三层滚动或压缩播放器才能成立，立即停止并重新设计。完成 M4.5 后，再为只读评论与相关推荐分别决定是否立项。
