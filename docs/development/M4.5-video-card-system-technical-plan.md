# M4.5 视频卡片视觉复用实施草案

> Revision：T4。状态：实施完成；本文件仍只记录技术方案，不代表用户逐项确认实施细节。完成证据见 [`../validation/M4.5-video-card-system-2026-07-24.md`](../validation/M4.5-video-card-system-2026-07-24.md)。

## 目标切片

**Goal**：热门、搜索和观看历史使用同一无业务语义的视频卡片外壳；历史改为卡片流并显示观看进度与观看时间。

**Context**：

- 首页／搜索当前使用 `GuestVideoCard` 与 `GuestVideoGridLayout`。
- 历史当前使用 `List` 与私有 `WatchHistoryRow`。
- `WatchHistoryItem.owner.avatarURL` 已是可选值；adapter 会把负进度归一化为总时长。

**Constraints**：

- Feature 之间禁止互相 import。
- `BiliUI` 不消费领域模型，不发请求，不持有 Task，不解释 API。
- 当前图片 URL 优化、卡片尺寸、两行标题和选择描边行为保持一致。
- 历史 ViewModel、分页、认证失效、取消和内存清理 owner 不变。

**Done when**：决策契约的最低证据全部满足，用户完成历史页视觉观察。

## 计划结构

```text
Packages/BiliKitCore/Sources/BiliUI/
├── VideoCard.swift
└── VideoCardGridLayout.swift
```

`VideoCard` 接收：

- 封面 URL、可选头像 URL、是否显示头像槽位、标题、选中状态；
- 零到多个封面左侧指标；
- 可选封面右侧文本；
- 底部左侧文本与可选右侧文本。

所有可选元素缺省时折叠，不保留固定槽位；头像槽位启用但 URL 缺失时使用人物占位图。图片加载失败继续使用非业务占位图。`BiliUI` 不识别 B 站图片主机或处理参数；调用方传入已经准备完成的 URL。

图片、渐变等装饰元素独立隐藏于辅助功能树；已经格式化的封面指标、底部元数据和选中状态参与卡片的组合朗读。选中卡片添加 `.isSelected` trait。

## Feature 适配

### Browse

- 保留 `GuestVideoCard` 作为 `PopularVideo`／`SearchVideo` 到显示文本的轻量 adapter。
- 继续由 Browse adapter 完成现有图片 URL 尺寸优化，再将封面、头像、标题和格式化结果传给 `BiliUI.VideoCard`。
- Browse 始终启用头像槽位，URL 缺失时保留当前人物占位图。
- 计数与发布日期格式化继续留在 Browse。
- 热门和搜索改用 `BiliUI.VideoCardGridLayout`。

### Library

- 用一个纵向 `ScrollView` 包含卡片 `LazyVGrid`；分页错误和加载更多按钮是网格之外的全宽 sibling，保留现有禁用态和辅助功能标识。
- `ForEach(items)` 继续使用 `WatchHistoryItem` 的 BVID identity，不引入 index、随机 UUID 或额外 `.id`。
- 新增只属于 History 的格式化器：
  - `progress >= duration && duration > 0` → “已看完”；
  - 其他情况 → `已看时长/总时长`；
  - 观看时间 → 今天／昨天 `HH:mm`，更早为 `M月d日 HH:mm`。
- 头像 URL 为 `nil` 时不显示头像。
- 卡片点击、分页、失败重试和 route deactivate 仍调用现有 ViewModel／闭包。

## 验证

- `Package.swift` 只新增内部 target 和两条 Feature dependency；共享类型使用 `package` 访问级别，工程脚本断言不存在 `BiliUI` library product。
- Browse 格式化既有测试保持通过。
- Library 新增进度、长时长、已看完、零时长和观看时间格式化测试；日期测试固定 Gregorian calendar 与时区。
- 既有 `WatchHistoryViewModelTests` 与 App chrome／路由／播放器生命周期测试保持通过。
- `sh Scripts/run-quality-gates.sh app`。
- 默认窗口和全屏下观察热门、搜索、历史的卡片比例、列数、可省略头像和分页入口。
- 最小真实 VoiceOver 路径确认历史进行中、已看完及加载更多可读；不建设自动辅助功能框架。
