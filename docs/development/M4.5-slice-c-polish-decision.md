# M4.5 Slice C 系统显示与辅助交互收口决策

> Revision：C1。状态：M4.5 生产契约 P7 已由用户确认，Slice A／B 已分别于
> `22bfa24`／`40862d0` 关闭；用户于 2026-07-24 明确开始 Slice C。本文件
> 只把既有授权收窄为最终精修与 Gate，不增加产品能力。

## Goal / Context / Constraints / Done when

**Goal**：让 M4.5 已有的搜索、热门、历史、账户与播放闭环在 macOS 系统显示、
键盘和主要辅助功能环境下保持可理解、可达且视觉稳定，并形成无个人数据的可重复
验证路径。

**Context**：Slice A 已完成两栏导航和共享视频卡片，Slice B 已完成稳定单
player host 的响应式播放页。当前真实状态已经覆盖 loading、empty、failure、
retry、signed-out、selection 与播放准备；主要缺口是卡片固定字号和不一致的
hover／pressed 表面、少量辅助语义，以及 XCUI 仍依赖现场网络。

**Constraints**：

- 不新增 endpoint、业务状态、持久化、target、Package、第三方 UI 依赖或
  通用 Design System。
- 不改变 route、播放 engine、renderer、subtitle／danmaku owner、媒体时间轴
  或资源清理。
- 不实现 M5 的完整 Commands、全量 VoiceOver 导航、独立播放器窗口、PiP、
  媒体键或窗口恢复。
- 只呈现已有真实状态；不得为了截图增加 offline、成功 toast、假评论、假推荐
  或账号资料。
- UI fixture 只在 Debug 且显式 launch argument 下启用，仍组装生产
  `ContentView`／`AppShellView`／Feature ViewModel；不能成为 Release 路径，
  也不能用来证明真实播放、签名、账号或网络行为。

**Done when**：

- 视频卡片使用语义字号，并在 hover、pressed、selected、键盘 focus 与
  Differentiate Without Color 下有可理解且不过度动画的反馈；Reduce Motion
  下不执行非必要缩放动画。
- 搜索框、卡片、账户、历史登录／重试、播放返回、字幕／弹幕与分 P 等主要控件
  具有可理解标签、状态或提示；搜索 Return 与播放页 Escape 返回可用。
- 自制无 URL、无真实 BVID、无账号内容的 Debug fixture 能覆盖热门、搜索、
  未登录历史、账户 sheet 与多分 P 播放页；浅／深色和文字放大使用可控 fixture
  成对验证。只读系统输入 Increase Contrast、Reduce Transparency、Reduce
  Motion 与 Differentiate Without Color 使用代码策略测试加真实系统观察，
  不通过私有设置或假环境值伪造。
- 1080×680、1320×820、自由缩放与全屏没有截断、遮挡、不可辨识选中态或
  不可达操作；App Gate、无个人数据 XCUI、独立黄区审查和用户最终关键路径通过。

## 最小实施范围

1. 在现有 `BiliUI` 内增加一个只服务 `VideoCard` 两个真实调用方的 package
   button style；它只处理 hover、pressed、selection、对比度与 Reduce Motion，
   不扩张成通用按钮库。
2. 把卡片固定点数字号替换为视觉接近的系统语义字号；保持 16:9、两行标题、
   网格密度和现有内容语义。
3. 为 AppKit 搜索框、播放返回、分 P 行、登录二维码与失败重试补齐必要的
   accessibility label／hint／identifier；不隐藏系统原生控件语义。
4. 在 Debug-only Composition 文件中组装最小内存 fixture adapters。假值使用
   `fixture-video-*` 等明确非 BVID identity，不包含 URL、个人账号、字幕或
   弹幕正文；Release 构建始终使用 `.live()`。
5. XCUI 只走一条核心路径并做一组成对显示检查，不建立截图服务、manifest、
   像素 diff、全矩阵 runner 或性能 harness。

## 证据、停止与回退

- 纯策略测试固定卡片交互反馈在 hover／pressed／selected／increased contrast
  与 Reduce Motion 下的结果；SwiftUI 像素外观由 fixture App 观察，不伪装成
  snapshot 单测。
- XCUI 固定“热门 → 搜索 Return → 结果 → 播放 → Escape 返回 → 未登录历史
  → 账户 sheet”及默认／辅助显示两种 fixture 配置。
- 最终只运行最高适用 `app` Gate；若 engine／renderer 未触碰且关键路径资源
  正常，不重跑 M4 30 分钟 probe。
- Slice C 完成后由新上下文 reviewer 检查状态真实性、键盘／辅助语义、
  fixture 隔离、Release 边界和维护性，再做价值删减与可理解性复核。

若卡片反馈需要自定义焦点引擎、fixture 需要复制 Feature 状态机或媒体 owner、
系统显示支持需要全局主题框架，立即停止并保留系统默认行为。回退时删除新增
style／fixture，恢复 Slice B 已通过 Gate 的界面，不影响 route 或播放。
