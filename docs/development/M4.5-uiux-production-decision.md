# M4.5 UI/UX 生产决策契约

> Revision：P7。状态：用户已于 2026-07-23 确认三项授权；M4 前置 Gate 已由 PR #7 的 macOS 15/26 CI 满足。Slice A 首页／搜索与导航外壳已通过本机 Gate 和用户视觉观察，并提交为 `20f3322`。用户随后授权以同一基础卡片视觉迁移观看历史；该窄例外见 [`M4.5-video-card-system-decision.md`](./M4.5-video-card-system-decision.md) 与 [ADR 0009](../adr/0009-narrow-biliui-video-card-boundary.md)。已确认的设计方向见 [`M4.5-uiux-decision.md`](./M4.5-uiux-decision.md)，长期产品机会见 [`../product/UIUX-VISION.md`](../product/UIUX-VISION.md)。

## 通俗摘要

M4.5 把当前验证用的三栏界面改成适合日常使用的 Mac 两栏 App：搜索置顶，热门和历史成为主页面，账户固定在侧栏底部；热门与搜索使用 16:9 卡片流；视频在同一窗口进入独立播放页。只重排已有能力并隐藏单分 P，不新增评论、推荐、关注、搜索历史或服务端接口。窗口缩放、返回或换视频不能造成重启或重复播放，退出后不能遗留后台播放资源。

M4 已正式关闭，本轮是生产迁移，不新开或合入播放实验；既有 M4 性能与清理证据只作为回归基线。

## 本轮只支持两个决策

1. **产品结构**：采用“App 侧栏 + 主内容”的两栏外壳，以及独立播放页、响应式视频卡片和已确认的信息层级。
2. **安全迁移**：由 App 层唯一持有一级 route 和内存级返回上下文；布局变化不复制、替换或重新加载播放器 host。

本轮不比较多套视觉风格，也不建立通用设计系统。Apple TV 和 B 站只提供导航习惯与信息层级参考，不复制像素、素材或网页组件。

## 获得授权的范围

- 侧栏顺序固定为搜索、热门、观看历史；账户入口固定在左下。认证仍是短生命周期 sheet，历史不再是 sheet。
- 热门与搜索复用同一 16:9 响应式卡片呈现；标题最多两行，UP 主、播放量、时长等只使用现有模型字段。
- 搜索框进入搜索页 toolbar；空页保持简洁。分类 Tab、筛选、排序、搜索历史、热搜和分区内容不在本轮。
- 未登录进入历史页时显示内联登录引导，用户主动点击后才复用现有认证 sheet；登录后才加载历史。
- 视频在同一窗口进入独立播放页；返回时在窗口内恢复来源页面、搜索词和选中卡片。仅当现有容器能用稳定 identity 低成本恢复时保留精确滚动位置；否则先以不迷失来源为 Gate。
- 播放页主区按“标题—UP 主／播放量／弹幕数—播放器—字幕与弹幕设置—简介”排列。右栏当前只显示多分 P；单分 P 隐藏整个模块。现有多分 P 只展示列表与当前项，本轮不新增分 P 选择能力。
- 覆盖 1080×680、1320×820、自由缩放和全屏，以及浅色／深色、文字放大和系统辅助显示。

## 不变量与主要风险

- App shell 是唯一一级 route owner；Feature 之间不得直接 import，也不跨页面传递 ViewModel、Repository 或任意字典。
- 整个窗口只存在一个播放 engine、一个 player host 和一个弹幕 surface owner。宽窄切换只改变播放器旁边的右栏，不得触发 item reload、时间轴重启或第二个 surface。
- 卡片或历史选择只改变 App route，由 route 变化在唯一位置开始播放；返回也只在唯一位置停止详情、字幕和弹幕。快速返回后再选另一视频、登出、认证失效和关闭窗口时，旧结果不得回流，离开播放后资源仍能归零。
- 历史加载中离开后再次进入必须能恢复；登出、认证失效和关闭窗口清空已加载的个性化条目。认证失效只回到内联引导，不自动弹登录 sheet。
- 返回快照只存在内存；不新增 UserDefaults、SwiftData、schema 或后台恢复。登出和关闭窗口清除个性化来源快照。
- 不显示假评论、假推荐、假 UP 主资料或空壳模块；不新增 endpoint、写操作或隐藏的未来状态。

## 复杂度预算

- 分三段完成：完整可工作的两栏路由迁移、播放页响应式重排、精修与 Gate；一次只合入一个可验证切片，不保留新旧双导航。
- 最多一个 App route 类型、一个内存返回快照；除两个真实 Feature 调用方共同需要、且由 C2/ADR 0009 限定的 Package 内部 `BiliUI` target 外，零新 target、Package、endpoint、schema、业务 port、第三方 UI 依赖或通用布局框架。
- 只在两个以上真实页面复用时提取最小卡片或视觉 token；不创建 `DesignSystem`。
- 自动化只覆盖 route／取消／登出／单分 P／稳定 player host 等关键不变量；视觉矩阵只取两个既定窗口和必要系统模式，不做完整笛卡尔积。

## 最低证据、停止与下一步

完成至少需要：

- 首页、搜索、历史、账户、播放和返回形成无死路的真实闭环；单分 P 隐藏，多分 P 保持现有只读列表与当前项展示。
- 播放期间底层播放器视图 active 为 1，整条关键序列 peak 为 1；同一次播放的窄宽／全屏期间 identity 不变，返回或关闭后 active 为 0。进入另一视频可以获得新 identity。该证据只使用每个窗口独立、位于现有 host 创建／销毁边界的一处测试 hook，不建立全局或通用诊断框架。
- Package/App Gate、无个人数据 XCUI smoke、两个窗口的真实视觉检查、键盘／焦点／主要辅助功能检查和独立审查通过。
- M4 既有 40/80 events/s 与 30 分钟证据只作基线；只有触碰 engine／renderer，或关键迁移序列出现资源异常时才重跑既有 probe，不重复开发 benchmark framework。
- 实现和技术证据通过后，由用户亲自走一次“播放→宽窄／全屏→返回→选择另一视频→关闭”真实链路，确认播放连续、来源不迷失且没有双导航；无需理解内部计数。

若必须增加新数据能力、第二套导航、第二个播放器 owner、持久化、通用布局框架，或无法证明稳定单一 player host，立即停止并重新请求授权。完成 M4.5 后，评论、相关推荐、UP 主资料与搜索发现能力仍分别立项。

## 三项授权

1. 是否授权上述两栏导航、卡片流、独立播放页、信息层级、未登录历史引导和单分 P 隐藏作为 M4.5 生产范围？
2. 是否同意 App shell 唯一路由、单一播放器 host、仅内存返回上下文，以及不新增数据能力／持久化的边界？
3. 是否同意按三段小步实现，并在现有播放、字幕、弹幕或资源表现变差，或复杂度超预算时停止而不是继续堆机制？
