# M2 游客 API 核心链路验证——2026-07-21

## 结论

`BiliAPI` 数据层已具备接入 App 的最小游客链路：热门或搜索 → 视频详情 → 分 P → AVC/AAC 播放描述。匿名 WBI key 获取、签名、搜索解码和单次 key 刷新重试已由受控测试覆盖，并在真实网络中通过。

这一结论只表明 API 与无 UI 编排层已达到产品接入点。App composition root 和游客主流程状态模型已于同日接入，但 M2 尚未关闭；`NavigationSplitView`、可操作播放器与 UI 错误状态仍待完成。

## 环境与真实网络证据

- 验证日期：2026-07-21（Asia/Tokyo）
- 本地系统：macOS 26.5.2（Apple Silicon）
- 工具链：Xcode 26.6、Swift 6.3.3
- Package deployment target：macOS 15
- 身份：匿名，未使用 Cookie 或 token

同日重新核对的 endpoint：

| --- | --- | --- |
| 热门 | `/x/web-interface/popular` | 通过 |

## 受控回归证据

- Swift Package 测试 43 项全部通过。
- 固定 WBI 混排表、参数排序、`!'()*` 过滤、百分号编码、时间戳与 MD5 结果的确定性向量。
- 同一 UTC 日内复用 WBI key；获取或解析失败不会缓存成持久失败状态。
- 搜索遭 API `-403` 或 HTTP `403` 拒绝时，清理 key、重新获取、重签并且只重试一次。
- 脱敏 nav 和搜索 fixture 固定当前字段形状，并验证标题 HTML 标记去除、协议相对图片 URL 补全与时长解析。
- 已有回归同时覆盖 HTML 风控页、非零 API code、字段缺失、输入校验、取消传播和旧结果隔离。

## 边界与后续工作

- 真实搜索的 20 条原始 `video` 结果中，有 1 条没有可用 BVID。数据层会过滤这类无法打开的条目，但 `totalResults` 继续保留服务端原始计数，因此页内数量可能小于 `pageSize`。
- 搜索结果和 WBI key 是动态外部依赖；线上单次失败不能单独证明本地签名或解码回归。
- 当前没有对并发的首次搜索合并 nav 请求；UI 层仍应做输入防抖与取消。
- App 组装与状态模型的 4 项单元测试已覆盖热门加载、首分 P 播放请求、媒体请求头传递和快速切换隔离。
- 下一步是搭建最小 `NavigationSplitView`，让现有状态模型在真实产品界面中驱动热门、详情、分 P 和播放器。
