# M2.5 MVVM / Clean Architecture 验证记录

- 日期：2026-07-21（Asia/Tokyo）
- 本地系统：macOS 26.5.2（25F84），Apple Silicon arm64
- 最低 deployment target：macOS 15.0
- 范围：保持 M2 游客产品行为不变，验证分层、编译边界、状态竞争与真实 UI/播放接线

## 结论

M2.5 Gate 通过。游客流程已从 App 原型迁为 Domain、Application、adapter、Feature Presentation 与 composition root 的单向结构；旧的 `GuestAppModel` 和 `GuestVideoCoordinator` 已删除，没有保留双轨实现。

## 结构证据

- `BiliModels`：视频与播放 Domain 值。
- `BiliApplication`：`GuestContentRepository`、`PlaybackControlling`、Feed/视频准备 Use Case 和应用级错误。
- `BiliAPI`：endpoint DTO、WBI、原始 API 错误及 `BiliGuestRepository` adapter。
- `BiliPlayback`：M1 播放算法及实现 Application port 的 `AVPlayerEngine`。
- `BiliGuestFeature`：按 `Feed/`、`VideoDetail/`、`GuestScene/` 组织的 View、ViewModel 与 UI State。
- App target：`App/`、`Composition/`、`Platform/` 和 Assets。

`Scripts/check-architecture.sh` 已在本地通过并加入 CI，检查：

- `BiliModels` 不导入外层模块或 UI/播放框架。
- `BiliApplication` 不导入 API、Networking、Playback、Feature 或 UI/播放框架。
- `BiliGuestFeature` 不导入 API、Networking、Playback、AVFoundation/AVKit/AppKit。

模型与依赖决策见 [ADR 0004](../adr/0004-mvvm-clean-architecture.md)。

## 真实界面回归

使用本轮构建的 `/tmp/BiliKitMac-M25-derived/Build/Products/Debug/BiliKit.app` 完成：

1. 冷启动进入热门，真实列表和封面区域正常加载。
2. 切换搜索并提交 `macOS`，返回约 1,000 条结果，首屏 19 条可选择结果。
3. 打开首条“万年Windows用户换macOS后，给我难受坏了！”，详情、P1 和系统播放器出现。
4. 点击播放后时间轴从 0 前进到至少 9 秒，证明注入的 `AVPlayerView` 与新的播放 port 接线有效。
5. 切换回热门后详情与播放器被移除，空详情状态恢复；再回搜索时关键词和搜索意图保留并重新得到结果。

本次真实请求只证明验证时刻的链路；热门内容、搜索条数、样本可用性和 CDN 行为均可能随外部服务变化。
