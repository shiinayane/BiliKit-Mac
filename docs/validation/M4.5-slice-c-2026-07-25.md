- Fixture 只在 `#if DEBUG` 且显式 `-ui-testing` 时启用；孤立
  `-ui-testing-compact`、`-ui-testing-dark` 或

- Fixture 位于 Composition，仍组装生产 `ContentView`、App shell 与 Feature
  ViewModel；adapter 只返回固定内存假值。
- 假值没有 URL、真实 BVID、账号、历史、字幕／弹幕正文、Cookie、token、
  AVPlayer 或网络 client；集合上限固定为 8 个热门、4 个搜索和 3 个分 P。
- Release 构建不编译 fixture 分支，普通 Debug 与全部 Release 均使用
  `ContentView()` live composition。

## 最终 XCUI 复跑

当前 macOS 会话会恢复“应用运行但零窗口”的状态。XCUI 启动 helper 先等待
标准窗口，仅在确认没有窗口时激活 App 并发送 `Command-N`；正常首窗存在时
不会重复创建窗口。compact 用例使用滚轮增量把 disclosure 滚入可见区域，再
点击实际折叠箭头热区，避免把系统自动滚动或 label 点击误记为成功展开。

最终命令：

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild -project BiliKitMac.xcodeproj -scheme BiliKitMac \
  -configuration Debug -destination 'platform=macOS,arch=arm64' \
  -derivedDataPath /tmp/BiliKitMac-sliceC-xcui-final2-20260725 \
  build-for-testing -quiet

DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild -project BiliKitMac.xcodeproj -scheme BiliKitMac \
  -configuration Debug -destination 'platform=macOS,arch=arm64' \
  -derivedDataPath /tmp/BiliKitMac-sliceC-xcui-final2-20260725 \
  -resultBundlePath /tmp/BiliKitMac-sliceC-xcui-final2-20260725.xcresult \
  test-without-building \
  -only-testing:BiliKitMacUITests/BiliKitMacUITests
```

结果：2 项通过，0 项失败。审查补充的 selected trait 断言随后单独复跑
compact 用例，1 项通过、0 项失败。XCUI Escape 只证明 route 返回，资源归零
继续引用既有确定性 App／ViewModel lifecycle tests。

最终 xcresult 另记录一条未失败的 QoS runtime warning，观测点位于 Return
键的 XCUI event synthesis，发生在播放前。warning 栈未符号化，无法归属于
产品、test harness 或 XCTest／AppKit；红区补充审查未发现它与零窗口 fallback、
播放 stop/reset 或资源生命周期相连的证据，按 P3 保留观察。
