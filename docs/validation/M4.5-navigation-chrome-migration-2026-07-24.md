## 结论

A1 迁移的本地工程证据通过。`ContentView` 继续拥有窗口级模型、顶层任务、认证协调和清理；单一 `AppShellView.swift` 拥有 split view、侧栏、route switch 与四类页面 chrome。`AppNavigationModel`、播放／字幕／弹幕实现和 Feature 依赖方向未改变。

迁移尚未解除 M4.5 冻结。用户确认迁移后真实观感前，不进入 Slice B/C，也不把本记录写成完整 M4.5 Gate。

## 自动验证

运行：

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  sh Scripts/run-quality-gates.sh app
```

结果：

- 静态架构、秘密扫描、工程／entitlement／最低系统版本契约通过。
- Swift Package：181 个测试、31 个套件通过。
- App `build-for-testing` 与 App unit tests 通过。
- 签名 Keychain smoke 因当前 Gate 使用无签名测试宿主而按设计跳过。
- 需要显式账号配置的已登录字幕现场探针按设计跳过。

新增的最小证据：

- `WatchHistoryViewModel` 固定 `.loading`／`.loadingMore` 的 busy 投影；实际 toolbar 使用的 `HistoryRefreshButton` 直接绑定该值。
- 完全离线的 App route 测试托管真实 `ContentView`，覆盖播放 A → 返回 → 播放 B → 根 View 消失。事件为 create A、dismantle A、create B、dismantle B；每次 create/dismantle identity 配对，`peak == 1`，最终 active 为 0。
- 根 View 消失同时证明生产 `onDisappear` 被调用：route 回到热门、播放 stop 累计两次、视频 ViewModel 回到 idle。
- 测试认证和历史依赖均为合成 fake；不读取 Keychain、不访问网络、不接触个人数据。

## 签名 App 观察

当前签名 Debug App 在默认窗口和约 1080×680 的较窄窗口中完成无个人数据结构检查：

- 热门保留语义窗口标题，但 toolbar 中没有可见页面标题。
- 搜索保留语义窗口标题，toolbar 中没有可见页面标题，搜索框位于主内容区中央。
- 观看历史显示页面标题，刷新入口位于右侧操作区；窄窗口中仍可发现。
- 页面切换没有遗留、重复或错位的 toolbar item。

观察后仅补强测试 seam、离线 fake 与同一刷新按钮的 busy 投影，没有改变上述页面布局。

## 独立审查

- 价值／简化审查删除了一条被真实 route 序列完全覆盖的通用 host 测试，拒绝新增通用 probe、fixture framework 或更宽矩阵。
- 技术审查发现最初测试绕过 `ContentView.onDisappear`、chrome busy wiring 未固定以及测试误用了 live 认证；均以最小离线证据修正。
- 红区终审同时删除跨生命周期比较 `ObjectIdentifier` 地址的不可靠断言，并确认 playerContent 稳定构造、唯一 route／stop owner、任务与清理位置未漂移。

最终审查无 blocker。当前只剩用户对三个页面结果的迁移后视觉确认。
