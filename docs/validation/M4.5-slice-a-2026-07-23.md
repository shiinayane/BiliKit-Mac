## 路由与资源负向测试

- App route 测试固定热门／搜索／历史到播放的唯一副作用 owner、搜索词与来源恢复、快速 A→返回→B、同一播放不重复 load、返回／侧栏切换／窗口关闭只清理一次，以及认证失效清除个性化返回来源。
- 真实 `NSHostingView → PlayerHostView → NSViewRepresentable` 边界连续挂载两次，观察到 `active` 为 `1→0→1→0`，`peak == 1`。
- 播放准备失败的真实 `VideoDetailColumn` 宿主测试确认字幕与弹幕归零；同一视频 retry 后两者可以重新启动，避免失败页背后残留旧媒体 owner。
- 历史初次加载和分页中离开 route 都使用带一秒 deadline 的可观察请求／完成信号，确认迟到结果不能重新覆盖已离开的页面状态。

这些 hook 只统计 host active/peak，不记录内容 identity，也没有进入 `AppEnvironment`、生产日志或通用诊断框架。

## 真实 App smoke

1. 默认窗口显示两栏外壳；搜索位于侧栏最上方，热门与观看历史在其下，账户固定在左下；

5. 播放页返回、来源切换与 `Command-W` 关闭窗口均可到达；关闭后 App 进程仍存活，没有由关闭动作退出整个进程。

窗口在代码中固定 `.defaultSize(width: 1320, height: 820)` 与内容最小值 `1080×680`；真实 App 分别检查了默认窗口、缩放后的紧凑边界和当前显示器可用的大窗口，均未出现侧栏丢失、卡片不可达或双导航。Computer Use 截图会按窗口状态缩放，不能作为精确 frame 像素计量，因此这里只把静态尺寸契约与真实视觉结果合并使用，不声称截图本身证明了精确数值。
