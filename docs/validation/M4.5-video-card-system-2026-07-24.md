## 架构与实现证据

- `BiliUI` 是 `BiliKitCore` 内部 target，没有 library product，也不依赖领域、Application、Feature 或 adapter。
- `BiliBrowseFeature` 与 `BiliLibraryFeature` 只向卡片传入 URL、已格式化文本和选择状态；计数、发布日期、观看进度与“已看完”仍由各自 Feature 解释。
- Browse 在头像 URL 缺失时保留既有占位；History 在服务端没有头像时折叠头像槽位。
- 封面尺寸由独立的 16:9 底层容器决定；远端图片只作为 overlay 居中裁切，不能用自身固有比例反向改变卡片高度。
- 历史 ViewModel、认证、分页、generation、route deactivate 与个性化内存清理 owner 未移动。

## 签名 App 观察

在本机开发签名 App 中检查默认窗口和全屏：

- 三个页面保持同一圆角、排版、间距和响应式网格；大窗口约五列；

- 辅助功能树能读取进度／完成态、标题、UP 主和观看时间，装饰图片与渐变不重复朗读。
