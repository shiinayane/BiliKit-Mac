# M4.4 renderer 生产验证记录（2026-07-23）

## 环境与范围

- 日期：2026-07-23（Asia/Tokyo）
- 系统：macOS 26.5.2（25F84），Apple Silicon `arm64`
- 分支：`codex/m44-renderer-production-contract`
- 决策契约：P4；技术计划：PT4
- 数据：自制合成弹幕；记录不包含正文、视频标识、账号信息或凭据

本记录验证 v1 的 `CATextLayer + 内容级重墨` 生产路径、镜像覆盖模型、真实播放视觉效果和有界清理。它不证明其他 Mac、未来样式设置或高级弹幕的容量。

## 用户真实播放结论

用户在签名 Xcode 运行的真实播放器中确认：

- 当前 100% 显示区域的视觉效果通过，字号、重墨和整体观感满意；
- 倍速、前进/后退 seek、滚动/顶部/底部切换可用；
- 字幕与弹幕可以同时正常显示。

用户同时澄清未来显示区域的正确语义：顶部/滚动从上缘覆盖，底部从下缘镜像覆盖；50% 在中线相接，75% 在中间相交，100% 完全重叠。当前不实现显示区域、密度、字号、透明度、粗细、颜色或高级弹幕设置。

本轮反馈没有单独记录 resize/fullscreen、多窗口和切换视频/关闭页面的逐项现场结果；这些路径仍由确定性生命周期测试与 App 集成测试覆盖，后续真实回归发现差异时必须重新打开 Gate。

## 自动化与独立审查

完整 App Gate：

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
BILIKIT_DERIVED_DATA_PATH=/tmp/BiliKitMac-m44-derived \
sh Scripts/run-quality-gates.sh app
```

结果：static、Swift Package、App build-for-testing 与 App unit tests 全部通过。无签名 Gate 中需要真实 Keychain 或已登录样本的测试按契约跳过。

镜像覆盖定向测试：

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
xcrun swift test --package-path Packages/BiliKitCore \
  --filter DanmakuLaneAllocatorTests
```

结果：10 项通过。50%/75%/100% 的 top/scroll 上锚定、bottom 下锚定、跨 mode 相交和同 mode 防碰撞均有确定性断言；非法覆盖比例 fail closed。

独立 Sol High 红区只读审查结论：P4/PT4 delta `closed`，无 blocker 或 improvement。审查确认 resize/zero 会清 allocator 并推进 renderer epoch，owner、generation、640 hard cap 与 stop/cleanup 路径没有被镜像覆盖修订削弱。
